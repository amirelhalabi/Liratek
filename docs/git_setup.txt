# Git Setup Instructions (Different Account)

To set up a Git repository for this project using a specific (different) Git account, follow these steps. These commands configure the user settings **locally** for this repository so it doesn't affect your global Git configuration.

### 1. Initialize the Repository
Run this in the project root:
```bash
git init
```

### 2. Configure the User for THIS Project Only
Replace the placeholders with the username and email of the account you want to use.
```bash
git config user.name "Your New Username"
git config user.email "your.new.email@example.com"
```
*Note: omitting the `--global` flag ensures these settings apply ONLY to this folder.*

### 3. Stage and Commit Your Files
```bash
git add .
git commit -m "Initial commit"
```

### 4. Add Your Remote Repository
First, create an empty repository on GitHub/GitLab/Bitbucket. Then, copy the URL (HTTPS is recommended) and run:
```bash
git remote add origin <YOUR_REPO_URL>
```

### 5. Push to the Remote
```bash
git branch -M main
git push -u origin main
```

### Authentication Tip
When you push, if you are using HTTPS, it will ask for your username and password.
*   **Username:** The username of your different account.
*   **Password:** If you are using GitHub, you must use a **Personal Access Token (Classic)** instead of your actual account password. 
    *   Go to GitHub Settings -> Developer Settings -> Personal access tokens -> Tokens (classic).
    *   Generate a new token with `repo` permissions.
    *   Paste this token when the terminal asks for your password.
