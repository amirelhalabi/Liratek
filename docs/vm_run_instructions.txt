# Running Mac-Built Electron Windows App on a VM

Since you are developing on a Mac but targeting Windows, you cannot "run" the `.exe` directly on macOS. You need a virtualization layer.

## Option 1: Parallels Desktop (Recommended for Performance)
1.  **Install Parallels Desktop:** (Paid software, but easiest integration).
2.  **Install Windows 11 ARM:** Parallels will download and install this automatically with one click.
3.  **Setup Shared Folders:**
    *   Parallels usually shares your Mac `Documents` folder with Windows automatically.
    *   Your project code at `/Users/amir/Documents/liratek` will be accessible in Windows at `\\Mac\Home\Documents\liratek` or `Z:\Documents\liratek`.

## Option 2: UTM (Free / Open Source)
1.  **Download UTM:** https://mac.getutm.app/
2.  **Download Windows 11 ARM ISO:** You may need to fetch a VHDX image from the Microsoft Insider Preview (ARM64) or use the "CrystalFetch" tool inside UTM.
3.  **Install & Setup:** Follow UTM instructions to create a Windows VM.
4.  **Spice Tools:** Install "SPICE Guest Tools" in the Windows VM to enable copy/paste and shared folders.

## Development Workflow

### Method A: Develop on Mac, Build on Mac (Cross-Compile)
You can build a Windows `.exe` *from* your Mac, then copy it to the VM to test.
1.  **On Mac Terminal:**
    ```bash
    # Install Wine (required for some Windows build targets on Mac, though nsis often works without it)
    brew install wine-stable 

    # Run the build command
    npm run make -- --platform=win32 --arch=x64
    # OR if using electron-builder explicitly:
    electron-builder build --win --x64
    ```
2.  **Transfer:** Move the resulting `.exe` (in `out/make` or `dist/`) to the Windows VM via the shared folder.
3.  **Test:** Double-click the `.exe` in Windows.

### Method B: Develop on Mac, Run on Windows (Hot Reload) - **BEST for Layout Testing**
To test OS-specific features (like printing or file system paths) live:
1.  **On Mac:** Keep your code in `/Users/amir/Documents/liratek`.
2.  **On Windows VM:**
    *   Install **Node.js**: https://nodejs.org/ (Download Windows Installer).
    *   Install **Git**: https://git-scm.com/download/win.
    *   Open PowerShell or Command Prompt.
    *   Navigate to the shared folder:
        `cd Z:\Documents\liratek` (Parallels)
    *   Install dependencies (if `node_modules` causes issues across OS, delete it and run `npm install` *inside* Windows).
        *   *Tip:* It's often safer to copy the project folder to the Windows Desktop (`C:\Users\User\Desktop\liratek`) to avoid "Shared Folder" permission/symlink issues with `node_modules`.
    *   Run the app:
        `npm start`
3.  **Result:** The Electron app opens inside Windows with Hot Reloading. You edit code on Mac, save, and the Windows app updates instantly.

## Final Production Build
Always perform the **Final "Gold" Build** on the Windows VM itself to ensure native dependencies (like `better-sqlite3`) are compiled correctly for Windows.
1.  Copy source code to Windows VM.
2.  Run `npm install`.
3.  Run `npm run make` / `npm run build`.
4.  Take the output `.setup.exe` and distribute it.
